# 사용법 — FSS / PIPC 제재정보 일일 스크래퍼

## 개요

이 프로그램은 아래 3개 사이트에서 매일 오전 6시(한국시간)에 신규 제재 공시를 자동 수집하고,
Excel 파일과 원본 첨부 파일을 이메일로 발송합니다.

| 사이트 | 수집 내용 |
|--------|-----------|
| 금융감독원 검사결과제재 | 번호, 제재대상기관, 제재조치요구일, 내용, 관련부서 |
| 금융감독원 경영유의사항 공시 | 일련번호, 제재대상기관, 제재조치요구일, 내용, 관련부서 |
| 개인정보보호위원회 의결·결정 | 번호, 회의구분, 제목, 의결일 |

---

## 1단계 — Gmail 앱 비밀번호 발급

> 일반 Gmail 비밀번호는 사용 불가. 반드시 **앱 비밀번호**를 발급해야 합니다.

1. **Google 계정 보안 설정** 접속  
   → https://myaccount.google.com/security

2. **2단계 인증** 을 아직 켜지 않았다면 활성화

3. 검색창에 **"앱 비밀번호"** 입력 후 접속  
   (또는 2단계 인증 설정 페이지 하단에서 찾기)

4. 앱: **메일**, 기기: **Windows 컴퓨터** 선택 후 **생성**

5. 화면에 표시되는 **16자리 코드** 복사  
   예시: `abcd efgh ijkl mnop` → 저장 시 공백 제거: `abcdefghijklmnop`

---

## 2단계 — GitHub Secrets 등록 (자동 실행용)

저장소 페이지 → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| Secret 이름 | 설명 | 예시 |
|------------|------|------|
| `GMAIL_USER` | 이메일 발송에 사용할 Gmail 주소 | `sender@gmail.com` |
| `GMAIL_APP_PASSWORD` | 위에서 발급한 앱 비밀번호 (공백 제거) | `abcdefghijklmnop` |
| `RECIPIENT_EMAILS` | 수신자 이메일 (여러 명은 쉼표로 구분) | `a@mail.com,b@mail.com` |

등록 후 **Actions 탭 → "일일 제재정보 스크래핑 & 이메일 발송" → Run workflow** 로 즉시 테스트 가능.

---

## 3단계 — 자동 실행 스케줄 확인

`.github/workflows/daily_scrape.yml` 파일에 설정된 스케줄:

```
cron: "0 21 * * *"   ← UTC 기준 21:00 = 한국시간 오전 6:00
```

별도 조작 없이 매일 오전 6시에 자동 실행됩니다.

## 로컬 PC에서 직접 실행하기

### 환경 준비

```bash
# Python 3.11 이상 필요
python --version

# 가상환경 생성 (선택)
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Mac/Linux

# 의존성 설치
pip install -r requirements.txt
```

### 환경변수 설정

`.env.example` 파일을 복사해서 `.env` 로 저장한 뒤 내용 수정:

```
GMAIL_USER=sender@gmail.com
GMAIL_APP_PASSWORD=abcdefghijklmnop
RECIPIENT_EMAILS=receiver1@mail.com,receiver2@mail.com
```

### 실행

```bash
python main.py
```

실행이 완료되면 지정한 수신자에게 이메일이 발송됩니다.

## 수집 기간 변경 방법

기본값은 **전날 ~ 오늘** 입니다.

`main.py` 의 아래 줄을 수정하면 수집 기간을 바꿀 수 있습니다:

```python
# 기본: 전날부터
yesterday = now - timedelta(days=1)

# 예) 7일치로 변경
yesterday = now - timedelta(days=7)
```

## 문제 해결

| 증상 | 원인 및 해결 |
|------|-------------|
| 이메일이 안 옴 | GitHub Secrets 값 확인 / Gmail 앱 비밀번호 재발급 |
| `RECIPIENT_EMAILS` 오류 | Secret 이름 오타 확인 |
| Actions가 실행 안 됨 | 저장소 Settings → Actions → General → "Allow all actions" 확인 |
| 스크래핑 데이터가 비어 있음 | 사이트 구조 변경 가능 — Issues 탭에 제보 |
| 로컬 실행 시 인코딩 오류 | Python 3.11 이상 사용, `.env` 파일 UTF-8로 저장 |
